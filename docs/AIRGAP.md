# Руководство по развертыванию в изолированной сети (Air-Gapped)

Это руководство описывает процесс развертывания платформы Redroom в изолированной сетевой среде (air-gapped) без доступа к интернету.

## 1. Предварительные требования
- Машина подготовки (staging) с доступом в интернет (для загрузки образов и пакетов).
- Целевая машина в изолированной сети.
- Docker и Docker Compose установлены на обеих машинах.
- USB-накопитель или другой безопасный метод передачи данных.

## 2. Компоненты инфраструктуры
Для локальной работы Redroom требуются следующие службы:
- **PostgreSQL**: База данных для разведданных и пользовательских данных.
- **Node.js**: Сервер бэкенда и обслуживание фронтенда.
- **Ollama (Опционально, но рекомендуется)**: Для локальных возможностей ИИ (RAG / Ask AI).

## 3. Подготовка (Машина подготовки)

### A. Загрузка Docker-образов
```bash
docker pull postgres:16-alpine
docker pull node:20-alpine
docker pull ollama/ollama:latest
```

### B. Сохранение образов в архив
```bash
docker save postgres:16-alpine > postgres.tar
docker save node:20-alpine > node.tar
docker save ollama/ollama:latest > ollama.tar
```

### C. Загрузка NPM-зависимостей
Загрузите все зависимости проекта в папку vendor:
```bash
pnpm install --frozen-lockfile
tar -czf node_modules.tar.gz node_modules
```

## 4. Развертывание (Изолированная машина)

### A. Загрузка образов
```bash
docker load < postgres.tar
docker load < node.tar
docker load < ollama.tar
```

### B. Настройка локального ИИ (Ollama)
Функция «Ask AI» в Redroom настроена на работу с локальным ИИ в автономном режиме.
1. Запустите Ollama.
2. Загрузите модель (например, Llama3) на машине подготовки с помощью `ollama pull llama3` и экспортируйте её хранилище.
3. В файле `.env` Redroom установите:
```env
BUILT_IN_FORGE_API_URL=http://ollama:11434/v1
BUILT_IN_FORGE_API_KEY=ollama
```

### C. Запуск приложения
Используйте файл `docker-compose.yml`, который включает PostgreSQL и приложение Redroom. Поскольку внешнего интернета нет, убедитесь, что все сборщики данных (краулеры) либо отключены, либо направлены на внутренние зеркала.

## 5. Рекомендации по безопасности
- Используйте аппаратные модули безопасности (HSM) для управления ключами.
- Включите функцию экспорта в SIEM для передачи логов на внутренний экземпляр Splunk/QRadar в изолированной сети.
- Подключите секретные физические узлы данных через API «Внешние модули».
